# 0005 — 言語別 ISO のダウンロード冪等性

## 症状

multi-lang 構成 (en-us + ja-jp の golden が要る) を clean state からビルドすると、**ja-jp の golden
が作れない**。ログを追うと ja-jp ISO のダウンロードが「既にある」とスキップされ、en-us ISO から
ja-jp golden を焼こうとして整合が取れない。

## 原因

`Get-WindowsIso.ps1` の冪等チェックが、ISO を**ファイル名でなくパターン**で広く拾っていた
(`server.*2025` 等)。en-us ISO が既に置いてあると、ja-jp を要求しても「Server 2025 の ISO は
もうある」と誤判定してスキップしていた。**冪等性の粒度が間違っていた**典型。

## 対策

冪等判定を**この `$IsoName` の ISO があるか**に限定 (`Get-WindowsIso.ps1:56`)。別言語の ISO が
存在してもこの言語の取得はスキップしない:

```powershell
if (Test-Path $dest) {                       # $dest = IsoDir\$IsoName (言語別の具体名)
    if (Test-IsoValid $dest) { exit 0 }      # この言語の ISO が有効ならだけスキップ
    Remove-Item $dest                        # 壊れていれば取り直す
}
```

言語別 URL は fwlink の `clcid` / `culture` で切り替える (en-us=0x409 / ja-jp=0x411)。

## 派生して直した ISO まわりのハマり (同記事にまとめる)

- **`.downloading` のまま `Mount-DiskImage` して失敗**: 「仮想ディスク サポート プロバイダーが
  見つかりませんでした」。`Mount-DiskImage` は**拡張子で**プロバイダを決める。→ 先に最終 `.iso` 名へ
  `Move-Item` してから検証する (`Get-WindowsIso.ps1:80-81`)。
- **検証失敗時に 7-8GB を毎回削除していた** (ユーザー指摘「なんで ISO 消えてるの？」)。→ 検証に
  失敗しても**ファイルは残す**。次回起動の冪等チェックが再検証し、本当に壊れていればそこで作り直す。
  無駄な再ダウンロードを防ぐ (`Get-WindowsIso.ps1:87-89`)。
- 評価版は SHA256 ピンしない (MS が随時更新するため)。代わりに**マウントして install.wim/esd の
  有無**で妥当性検証する (`Test-IsoValid`)。

## 教訓 / 汎用ノウハウ

- **冪等チェックの「キー」を間違えると、足りないものを「ある」と誤判定して黙ってスキップする。**
  これは「失敗」より厄介 (エラーが出ない)。冪等の判定対象は、**生成物を一意に決める粒度**
  (ここでは言語を含むファイル名) に合わせる。
- **高価な成果物 (巨大ダウンロード) は、検証失敗で安易に消さない。** 残して再検証する方が、
  ネットワーク往復を無駄にしない。「壊れてたら消して再取得」は次回の冪等パスに任せる。
- `Mount-DiskImage` / `New-VHD` など、**拡張子でプロバイダや形式を決める** API は、一時ファイル名の
  ままでは動かない。確定名にしてから操作する。
